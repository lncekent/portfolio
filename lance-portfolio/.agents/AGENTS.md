# AGENTS.md — Lance Portfolio Project Rules

## Explain Before Acting

Before modifying any code, suggesting changes, or applying any fix, always follow
this structured teaching flow:

1. **Explain the problem** — What is the issue? Describe it clearly and simply.
2. **Explain why it matters** — Why does it need to be addressed? What are the consequences of ignoring it?
3. **Provide context** — Does this issue actually affect this specific project, or is it theoretical?
4. **Present solutions** — List the available options to fix or address it.
5. **Let the user decide** — Do NOT apply any solution until the user explicitly chooses one.

This rule exists because the user is learning and wants to understand problems deeply,
not just receive changes. Understanding the "why" is more valuable than the fix itself.

> Exception: Trivial typos or minor formatting fixes that the user has already identified
> themselves may be applied directly if the user explicitly asks to "just fix it."
